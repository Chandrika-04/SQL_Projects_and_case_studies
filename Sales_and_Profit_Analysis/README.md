# 📊Sales & Profit Analysis

> **Domain:** Business Intelligence / Retail Analytics  
> **Database:** Relational (3-table schema)  
> **Focus:** Sales performance, profitability, marketing spend, and inventory analysis

A hands-on SQL case study simulating the role of a **database administrator** analyzing retail customer and sales data. The project covers a wide range of SQL skills — from basic aggregations and filtering to stored procedures, user-defined functions, window functions, transactions, and set operations.

---

## 📁 Dataset

Three relational tables form the core of this case study:

### FactTable
`4200 rows × 14 columns`

| Column | Description |
|---|---|
| Date | Transaction date |
| ProductID | Foreign key linking to ProductTable |
| Profit | Profit earned on the transaction |
| Sales | Revenue from the sale |
| Margin | Profit margin |
| COGS | Cost of Goods Sold |
| Total Expenses | Total cost incurred |
| Marketing | Marketing spend allocated |
| Inventory | Inventory level |
| Budget Profit | Projected profit |
| Budget COGS | Projected COGS |
| Budget Margin | Projected margin |
| Budget Sales | Projected sales |
| Area Code | Foreign key linking to LocationTable |

### ProductTable
`13 rows × 4 columns`

| Column | Description |
|---|---|
| ProductID | Unique product identifier |
| Product | Product name |
| Product Type | High-level category (e.g., Coffee, Tea) |
| Type | Sub-classification (e.g., Regular) |

### LocationTable
`156 rows × 4 columns`

| Column | Description |
|---|---|
| Area Code | Unique area identifier |
| State | US state name |
| Market | Market region |
| Market Size | Size classification of the market |

## ✅ Tasks Covered

### 🔍 Basic Queries & Aggregations
- Count distinct states present in the LocationTable
- Count products of `Regular` type
- Find total marketing spend for a specific product (ProductID = 1)
- Find minimum sales across all products
- Find maximum COGS across all records

### 🔎 Filtering & Conditions
- Display all products where product type is `Coffee`
- Display records where total expenses exceed 40
- Display records where total expenses are between 100 and 200
- Display date, ProductID, and sales for a specific expense range

### 📐 Aggregations by Group
- Calculate average sales for area code 719
- Find total profit generated by Colorado
- Compute average inventory for each ProductID
- Calculate average total expense per ProductID per date
- Calculate average budget margin per product (filtered: average > 100)
- Find total sales on a specific date (`2010-01-01`)

### 🗂 Sorting & Ordering
- Display states in sequential (alphabetical) order from LocationTable

### 🔗 Joins & Multi-table Queries
- Display a combined view with: `Date`, `ProductID`, `Product_Type`, `Product`, `Sales`, `Profit`, `State`, `Area_Code`
- Find state-wise profit and sales
- Find state-wise profit and sales along with product name

### 🏆 Window Functions
- Rank products by sales using **dense rank** (no gaps in ranking)

### 📈 Calculated Columns
- Calculate a **5% increase in sales** for each record
- Find the maximum profit along with its corresponding ProductID and Product Type

### ⚙️ Stored Procedures & Functions
- Create a **stored procedure** to fetch records from ProductTable filtered by product type
- Create a **user-defined function (UDF)** to retrieve a specific product type based on user input

### 🔄 Transactions
- Update product type from `Coffee` to `Tea` for ProductID = 1, then **roll back** the change using a transaction

### 📅 Rollup & Grouping Sets
- Generate total weekly sales with `Date` and `ProductID` using **ROLLUP** for hierarchical subtotals

### 🔀 Set Operations
- Apply **UNION** and **INTERSECT** on tables sharing the `Area Code` attribute

### 🧠 CASE / Conditional Logic
- Label records as `Profit` or `Loss` based on whether total expenses fall below or exceed 60

### 🗑️ DML Operations
- Delete all records from ProductTable where product type is `Regular`

### 🔤 String Functions
- Display the **ASCII value** of the 5th character from the `Product` column

## 🛠 How to Run

1. Set up a **MySQL** (or compatible SQL) environment.
2. Create the three tables — `FactTable`, `ProductTable`, `LocationTable` — and import the data.
3. Execute the queries in logical order (basic → advanced) as listed in the Tasks section above.
4. For stored procedures and UDFs, ensure your client supports multi-statement execution (use `DELIMITER $$` in MySQL Workbench if needed).
5. For the transaction task, run inside an explicit transaction block and use `ROLLBACK` to undo changes safely.
